### 2.2 Focal loss

- **γ (gamma)**: `mention_focal_gamma = 2.0` (config default; not overridden).
- **Formula**: `focal = ((1 - p_t)^γ) * cross_entropy(logits, labels, weight=class_weights, reduction='none')`, mean-reduced. See `FocalTokenClassifier.forward` at [src/pestclef/modernbert.py:626–634](src/pestclef/modernbert.py:626).
- **`ignore_index = -100`**: subword-piece labels beyond the first token per word are masked to `-100` and excluded from the loss.

### 2.3 Class-weight cap (12.0)

Class weights are computed by `_build_mention_class_weights` at [src/pestclef/modernbert.py:579](src/pestclef/modernbert.py:579):

1. Count each label's occurrences across all training rows (valid labels only, excluding `-100`).
2. Compute frequencies `f_i = count_i / total`.
3. Raw weights: `w_i = 1 / max(f_i, 1e-6)`.
4. Normalize so that mean(w) = 1.
5. Clamp: `w_i = min(w_i, cap=12.0)`.
6. Force `w_O = min(w_O, 1.0)` — the outside label cannot exceed weight 1.

The cap prevents runaway focal weighting on rare labels (Vector, Transmits) while preserving proportional up-weighting.

### 2.4 Sliding window over document text

- **Token limit**: `max_length=1024` (v14 default). Applied by the HuggingFace tokenizer via `truncation=True, max_length=1024, return_overflowing_tokens=True`.
- **Stride**: `doc_stride=256`. Overlap between consecutive windows is 256 tokens.
- **Padding**: `padding="max_length"` (every window is right-padded to 1024).

At inference, each window is decoded independently by `decode_window_predictions`, then spans are merged by `combine_span_candidates`.

### 2.5 Lexicon construction rule

`MentionLexicon.from_documents` at [src/pestclef/mention_detection.py:52](src/pestclef/mention_detection.py:52):

- **Input**: the training document set (via `mention_lexicon = MentionLexicon.from_documents(train_documents, config)`).
- **Aliases source**: every canonical entity's `alias_forms` (all gold surface forms seen in training) *and* the entity's `canonical_form` (normalized).
- **Minimum alias length**: `min_alias_length = 3` characters (after casefolding and whitespace normalization).
- **Per-alias type assignment**: if the same alias string is observed with multiple entity types across the corpus, it is assigned to the **most frequent** type via `Counter.most_common(1)`. No frequency threshold — a single occurrence is enough for an alias to enter the lexicon (subject to the length filter).
- **Global, not per-type**: `alias_counts: Dict[alias, Counter[type]]` is one flat dict keyed by alias; type conflicts are resolved by argmax at lookup time.

Lexicon match at inference is via **Aho-Corasick multi-pattern automaton** (built once, uses `pyahocorasick`; falls back to per-alias regex if `pyahocorasick` unavailable). Word-boundary check is enforced manually — the character before the match must not be alphanumeric (`\w`), and the character after must not be either.

### 2.6 Lexicon confidence 0.55

This is a **fixed floor**, not a computed value. Set in `ExperimentConfig.mention_hybrid_lexicon_confidence = 0.55`. When the hybrid blender processes a lexicon-derived candidate span, it computes:

```python
confidence = max(base_confidence(=0.55), external_confidence(=0.0), threshold + 0.05)
```

(at [src/pestclef/modernbert.py:922–925](src/pestclef/modernbert.py:922)). The 0.05 above the per-type threshold ensures lexicon matches always survive the per-type threshold filter. The 0.55 was chosen empirically before the threshold-tuning sweep; no calibration procedure produced it. `external_confidence` is 0.0 for train-derived lexicon entries (only becomes nonzero for the EPPO Bayer lexicon in the v21b experiments, which the winning submission does not use).

### 2.7 Blending rule when lexicon and classifier disagree

**Rule**: for each unique `(entity_type, start_offset, end_offset)` triple, keep the highest-confidence span. Implemented in `combine_span_candidates` at [src/pestclef/modernbert.py:831–841](src/pestclef/modernbert.py:831):

```python
best_by_span: Dict[tuple[str, int, int], PredictedMentionSpan] = {}
for spans in span_groups:
    for span in spans:
        key = (span.entity_type, span.start, span.end)
        existing = best_by_span.get(key)
        if existing is None or span.confidence > existing.confidence:
            best_by_span[key] = span
```

- When the same span offsets get identical type from both sources: neural (typical 0.7–0.9 softmax) wins over lexicon (0.55 floor).
- When the same span offsets get *different* entity types (e.g. neural predicts "Pest" at [10,20] but lexicon predicts "Vector"): they are treated as **different keys** and both survive the initial blend, then `merge_predicted_mentions` (§2.9) picks a winner via `_span_sort_score` in a greedy overlap-elimination pass.

### 2.8 Denylist source and size

Hand-curated, hard-coded in `TYPE_DENYLISTS` at [src/pestclef/modernbert.py:114–155](src/pestclef/modernbert.py:114). Per-type entry counts:

| Type | Denylist size | Example entries |
|---|---|---|

| Location | 9 | `green`, `red`, `supplementary`, `latest`, `faostat`, `territory of the union`, `entire provinces`, `individuals`, `farmers` |
| Vector | 13 | `individuals`, `individual`, `orf`, `supplementary`, `red`, `immunoglobulin-like`, `5-10`, `genbank`, `vector`, `vectors`, `species`, `pathotype`, `st2` |

No denylist for Disease, Pest, Plant. Denylist matching uses `normalize_text` (casefold + whitespace-collapse). Applied by `_passes_type_specific_cleanup` at [src/pestclef/modernbert.py:947](src/pestclef/modernbert.py:947) when `mention_type_denylists_enabled=True` (default in v14 config).

### 2.9 Per-type confidence threshold grid searched

Per-type threshold tuning is done by `tune_mention_thresholds` at [src/pestclef/modernbert.py:1150](src/pestclef/modernbert.py:1150). **The grid is not fixed — it is data-adaptive**:

1. Collect the set of unique `span.confidence` values that appear in dev raw span predictions for the target entity type (rounded to 4 decimals). Always include 0.0.
2. If the resulting grid has > 25 points, sub-sample to 25 uniformly-spaced indices via `np.linspace(0, N-1, 25).astype(int)` and add 0.0 back if it was dropped. This compression is `_compress_threshold_grid` at [src/pestclef/modernbert.py:1239](src/pestclef/modernbert.py:1239).
3. For each threshold in the grid, apply it to the target type, re-merge candidate spans, and compute the objective.

**Objective (v14, "relation_aware_v1" strategy)**: weighted combination of candidate recall and mention F1:

```python
score = 0.75 * candidate_recall + 0.25 * mention_f1
```

(weights are `mention_threshold_candidate_recall_weight=0.75` and `mention_threshold_mention_f1_weight=0.25`; from `config.py`.) Candidate recall is measured after enumerating relation-candidate pairs on predicted mentions and comparing to gold relation edges.

**Tie-breaking** (when scores are within `1e-9`): prefer higher candidate recall → higher F1 → higher recall → lower threshold. See [src/pestclef/modernbert.py:1217–1228](src/pestclef/modernbert.py:1217).

## 3. Stage 2 — Relation classification

### 3.1 Marker-token format

Defined in `get_relation_marker_tokens` at [src/pestclef/modernbert.py:192](src/pestclef/modernbert.py:192) and inserted by `build_relation_context_text` at [src/pestclef/modernbert.py:1416–1422](src/pestclef/modernbert.py:1416):

- **Open**: `<SUBJ_<TYPE>>` and `<OBJ_<TYPE>>`
- **Close**: `</SUBJ_<TYPE>>` and `</OBJ_<TYPE>>`
- **Type** is the entity type (e.g., `Pest`, `Location`), so the actual tokens are `<SUBJ_Pest>`, `</SUBJ_Pest>`, `<OBJ_Location>`, `</OBJ_Location>`, etc.

Each marker is **space-padded** (`f" <SUBJ_{...}> "`) on insertion to avoid tokenization glueing them to adjacent text. Inserts are applied in reverse position order (largest-offset first) so earlier offsets remain valid. The 28 marker tokens (7 types × 4 markers each) are added to the tokenizer's special-tokens vocabulary before training (via `add_special_tokens`), and the embedding matrix is resized with `mean_resizing=True` (the default in Transformers 5.x).

Example resulting text snippet:

```
"a novel outbreak of <SUBJ_Pest> ToBRFV </SUBJ_Pest> has been reported in <OBJ_Location> Finland </OBJ_Location> ..."
```

### 3.2 Context window definition ("±2 sentences") and sentence segmentation

**Radius**: `relation_context_sentence_radius = 2` (v14 default).

**Selection logic** (in `build_relation_context_text`, [src/pestclef/modernbert.py:1404–1408](src/pestclef/modernbert.py:1404)):

```python
start_sentence = max(0, min(subj.sentence_index, obj.sentence_index) - 2)
end_sentence   = min(len(sentence_spans) - 1, max(subj.sentence_index, obj.sentence_index) + 2)
snippet        = text[sentence_spans[start_sentence][0] : sentence_spans[end_sentence][1]]
```

**Sentence segmentation**: **regex only, no spaCy or nltk**. `split_sentences` at [src/pestclef/data.py:216](src/pestclef/data.py:216) applies:

```python
SENTENCE_SPLIT = re.compile(r"(?<=[.!?])\s+")
```

— splits on whitespace immediately following `.`, `!`, or `?`. No abbreviation handling (so "sp." or "et al." create false splits). This was a deliberate simplicity choice; a spaCy-based segmenter was considered but not adopted because the added dependency wasn't justified by measured gains on dev.

### 3.3 Layout-block definition and `≤6 layout blocks` pruning distance

**Source**: layout blocks are **pre-annotated in the source dataset**. Each document JSON in `data/json/PestCLEF-2026_{split}.json` has a `layout` field: a list of `{tag: str, offsets: [start_char, end_char]}` entries reflecting HTML-like structure inferred by the shared-task organizers (e.g., `{"tag": "h1", "offsets": [0, 83]}`). We do not produce these; we consume them.

**Layout index of a mention**: the block whose `offsets` span contains the mention's character span. Computed by `locate_span` at [src/pestclef/data.py:231](src/pestclef/data.py:231); falls back to the closest block if no strict containment.

**Layout gap between two canonical entities**: minimum absolute difference of layout indices across all `(subj_mention, obj_mention)` pairs. Formula in `min_distance` at [src/pestclef/features.py:171](src/pestclef/features.py:171):

```python
min(|a - b| for a in subject.layout_indices for b in object.layout_indices)
```

**Pruning threshold**: `max_layout_distance = 6` (config default). A candidate pair with `layout_gap > 6` is rejected by `should_consider_pair` at [src/pestclef/features.py:138](src/pestclef/features.py:138). The same function also enforces `sentence_gap ≤ max_sentence_distance = 3` and the schema type-compatibility check.

### 3.4 Candidate-pair enumeration

`enumerate_candidate_entity_pairs` at [src/pestclef/features.py:153](src/pestclef/features.py:153):

- **Within-document only**: no cross-document candidate pairs.
- **All ordered pairs**: `for subject in entities: for obj in entities` — both directions considered, so `(A, B)` and `(B, A)` are separate candidates.
- **Self-pairs excluded**: `if subject.entity_id == obj.entity_id: continue`.
- **Entities pre-sorted** by `earliest_start` before pair enumeration (for deterministic order).
- **Distance + schema pruning**: each pair passes through `should_consider_pair` (§3.3).

An additional `relation_pair_pruning_profile = "precision_v1"` layer (applied only when `model_name == "modernbert_staged"`) narrows the sentence/layout gaps further for `(Any, Date)` pairs (≤1 sentence, ≤2 layout) and `(Any, Location)` pairs (≤2 sentence, ≤4 layout). See [src/pestclef/features.py:177–204](src/pestclef/features.py:177).

### 3.5 Training data mix (50/50 gold/predicted — how predicted mentions are injected)

Controlled by `mix_relation_examples` at [src/pestclef/modernbert.py:1561](src/pestclef/modernbert.py:1561):

```python
def mix_relation_examples(gold_examples, predicted_examples, predicted_mix_ratio, random_seed):
    mixed = list(gold_examples)
    if not predicted_examples or predicted_mix_ratio <= 0.0:
        return mixed
    maximum_predicted = max(1, int(round(len(gold_examples) * predicted_mix_ratio)))
    if len(predicted_examples) <= maximum_predicted:
        mixed.extend(predicted_examples)
        return mixed
    rng = np.random.default_rng(random_seed)
    selected_indices = sorted(rng.choice(len(predicted_examples), size=maximum_predicted, replace=False).tolist())
    mixed.extend(predicted_examples[index] for index in selected_indices)
    return mixed
```

**Predicted examples** are generated by running the just-trained mention detector on the training documents themselves (see `_collect_predicted_entity_state` in `pipeline.py`), enumerating candidate pairs from those predicted entities, and labeling each pair by matching subject/object canonical forms against gold relation edges. So the "predicted" mix injects candidates whose entities were surfaced by the model, some of which won't match gold, thereby introducing training-time exposure to the noise distribution the classifier faces at test time.

With `predicted_mix_ratio = 0.5`: up to `0.5 × |gold_examples|` predicted examples are randomly subsampled (without replacement, seeded by `random_seed`) and appended to the gold set. The gold set is kept **in full** — the 50/50 label is nominal, not a partition.

**Note on the seed**: the mix subsampling uses the same seed as the config's `random_seed`, so the mix is reproducible per seed (and differs across the 5 ensemble seeds).

### 3.6 Inverse-frequency weight formula (relation-classifier `pos_weight`)

Computed at [src/pestclef/modernbert.py:1875–1876](src/pestclef/modernbert.py:1875) and mirrored for stage-1 of the two-stage path at 1909–1910:

```python
positive_rate = label_matrix.mean(axis=0)   # shape [n_labels], value in [0, 1]
pos_weight    = np.where(
    positive_rate > 0,
    (1.0 - positive_rate) / np.maximum(positive_rate, 1e-4),
    1.0,
)
```

For a relation with positive rate `p`, the weight is `(1 - p) / p` (the "odds inverse" of the positive class). Applied as `pos_weight` in `BCEWithLogitsLoss(pos_weight=...)`. Labels with zero positives in the training rows get weight 1.0 (a no-op safe default).

### 3.7 Per-relation threshold grid

Grid construction in `calibrate_threshold` at [src/pestclef/model.py:277–279](src/pestclef/model.py:277):

```python
candidates = np.unique(np.quantile(scores, np.linspace(0.6, 0.995, 30)))
candidates = np.concatenate((
    [scores.min()],
    candidates,
    [scores.max()],
    [clipped_default, min_threshold, max_threshold],
))
```

- **30 quantiles from 60th to 99.5th percentile of the score distribution**, plus `score.min()`, `score.max()`, default threshold, min_threshold, max_threshold.
- **Total grid size**: ≤ 34 unique thresholds per relation.
- **Range**: `[min_threshold, max_threshold]` per config. **v14 uses `[0.05, 0.95]`** (wide band). For rare labels (< 20 positives via `relation_threshold_min_positives_for_full_tuning`), the range is clipped to `[default - 0.15, default + 0.15]` where `default` is the pre-calibration hardcoded threshold.
- **Objective**: F1 per label. Tie-breaking: first threshold found is kept (ties do occur but rarely because quantile-derived thresholds are numerically distinct).

The grid is defined per label, per training run. It is not shared across labels or across seeds.

## 4. Ensemble

### 4.1 Five random seeds

**`[13, 42, 1337, 7, 2025]`** — hard-coded default in `V14SeedConfig` at [scripts/v14_5seed_ensemble.py:78](scripts/v14_5seed_ensemble.py:78). These are passed to `random_seed` in each seed's `ExperimentConfig` and used to seed:
- The Python and NumPy `random` states via HuggingFace `set_seed(random_seed)` (called inside HuggingFace's own model loading).
- The `np.random.default_rng(random_seed)` in `mix_relation_examples` (§3.5).
- The `LogisticRegression(random_state=random_seed)` for the sklearn baseline.
- The `sklearn` `class_weight` computation (deterministic given the same seeded data ordering).

No explicit `torch.manual_seed` is set in the pipeline. Empirically, MPS forward/backward passes are deterministic *given* fixed data ordering (the DataLoader is seeded by the HuggingFace `set_seed`), so we observe reproducible per-seed metrics.

### 4.2 Key normalisation rule for `(document, subject, object)`

**No normalization.** The aggregation key at [scripts/multi_seed_ensemble.py:160–166](scripts/multi_seed_ensemble.py:160) is:

```python
key = (
    str(row["doc_id"]),
    str(row.get("subject_type", "")),
    str(row["subject"]),           # subject.canonical_form
    str(row.get("object_type", "")),
    str(row["object"]),            # object.canonical_form
)
```

The `subject` and `object` fields are the `canonical_form` strings as they appear on the `CanonicalEntity` objects — **not casefolded, no punctuation stripping, no diacritic folding**. The canonical form is chosen by `build_canonical_entities` (in `data.py`) as the most frequent surface variant of an entity across its cluster of mentions, preserving the original casing, punctuation, and diacritics of that surface form.

Consequence: if seed A predicts an entity with canonical form `"ToBRFV virus"` and seed B predicts `"tobrfv virus"` for the semantically same entity, **these are distinct keys in the ensemble** and each is treated as a single-seed-only candidate. This is one of the reasons the seed-count histogram is heavier at the extremes than at the middle (see §5). A future refinement would be to fold the aggregation key through a shared canonicalizer.

### 4.3 Averaging formula

**Arithmetic mean of sigmoid outputs**, averaged only over seeds that produced the key. See [scripts/multi_seed_ensemble.py:188–192](scripts/multi_seed_ensemble.py:188):

```python
averaged[key] = {
    label: (sum(values) / len(values)) if values else 0.0
    for label, values in label_scores.items()
}
```

- No geometric mean, no max, no median.
- Missing seeds contribute nothing to the label's average (they are neither counted as 0 nor as 1). A key that appears in 3 of 5 seeds is averaged over those 3.
- With the winning submission's k≥4 filter, only keys present in ≥4 seeds enter the averaged output at all.

### 4.4 Tie-breaking when threshold is exactly met

**Inclusive**: at [scripts/multi_seed_ensemble.py:210](scripts/multi_seed_ensemble.py:210):

```python
if score >= float(thresholds.get(label, 0.5)):
    edges_by_doc[doc_id].append(RelationEdge(...))
```

A prediction whose averaged sigmoid score exactly equals the threshold is **kept**. In practice this is a measure-zero case with float64 arithmetic and doesn't affect any submission.

Post-emission deduplication is by exact string match on `(subject, predicate, object)` — see [scripts/multi_seed_ensemble.py:216–222](scripts/multi_seed_ensemble.py:216). Duplicates arising from schema-multiple-slot pairs are dropped, keeping the first-encountered instance.
